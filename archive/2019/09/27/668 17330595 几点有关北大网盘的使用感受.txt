几点有关北大网盘的使用感受

#p 21005167 RyanBern r.bern 2019-09-28 14:27:19


2019-09-27 00:18:01
昨天 pyq 被北大网盘刷屏了，于是就下载了个北大网盘玩了一下午，和同学一起也做了好多测试。学校为大家提供新的服务还是要支持一下。试完之后怎么说呢，很感谢学校的网盘服务，但总感觉有点没到自己的心理预期。新网盘体验有点怪怪的。
首先提醒一下：**截止到 2019 年 9 月 26 日学校并没有正式发布网盘可以使用的通知，不要被 pyq 转发的各种东西误导了。**一切还只是在测试阶段，所以重要的文件暂时也先**不要在网盘上放**了，以免出事。
**一、北大网盘定位**
首先没有理解错的话，这个网盘应该是北大和第三方公司进行合作搭的，基于的平台是 AnyShare ([https://www.eisoo.com](jump-to.php?url=https%3A%2F%2Fwww.eisoo.com))。北大应该主要负责准备一些服务器然后在上面跑这个平台，并实现和北大统一认证对接的功能（必要）。就平台本身的功能而言大多都是别人做好的，不太清楚计算中心是否会在其基础上进行二次开发，做一些新功能出来。个人感觉如果是企业版平台的话软件大概率闭源，更改已经做好的功能比较困难。其实基于现有框架搭建也没什么不好，清华的云盘也是基于另一框架 Seafile ([https://www.seafile.com/](jump-to.php?url=https%3A%2F%2Fwww.seafile.com%2F)) 进行搭建的。只是说同学们的一些**建议和期望可能要落空了**，毕竟软件是第三方公司做的，计算中心也不是专门搞平台开发的。
**PS：刚刚确认了一下，Seafile 初期源自清华实验室**，**难怪他们用。** ([https://www.seafile.com/about/](jump-to.php?url=https%3A%2F%2Fwww.seafile.com%2Fabout%2F))
然后就是各种功能。北大网盘应该更像是 Dropbox，OneDrive，Seafile 这种同步盘模式，和国内知名的百度网盘**有根本区别**。之前看见其他的讨论贴，可能多数人还是更习惯于百度网盘这种把网盘当做一个网络 U 盘的模式，但实际北大网盘的亮点在于同步，主要用途是多台计算机（例如博士生工位电脑和个人笔记本）以及多人之间的合作。当成网盘使也不是不可以，但是操作会很不爽。网页端不支持大文件（1 GB 以上）以及断点续传，同步客户端又要把需要上传的文件放在指定位置（设置里有一项开启时自动上传系统指定目录）。Seafile 平台出了一个类似于“挂载客户端”的东西，可以直接将服务器目录挂载到本地，这就解决了同一文件要存两次的问题，但北大网盘目测没找到这样的客户端。
**二、客户端**
客户端让我最不满意的是**没有 Linux 客户端**，这让多少 Linux 用户心凉了一截啊。Onedrive 这种微软家的不支持 Linux 就算了，Dropbox，Seafile 等平台都是支持 Linux 的。现在我都怀疑 AnyShare 是不是跑在一个 Windows 服务器上。Windows 的客户端直接绑定在资源管理器上，没有单独的界面。应该是想做一个不用打开网页也能**即时分享，还原历史版本之类**的功能。这个可能会很舒服吧，但在客户端模式下是不能直接在顶层空间创建目录的。
北大网盘的本地目录名字叫 ShareCache，这个名字其实真的挺误导人的。它起名叫 Cache 但实际**并不是 Cache**，本地删除东西之后网盘端的东西也就没了（可以在回收站找到）。这跟 Dropbox 之类的机制是一样的。不在本地的文件它也会显示出来（左下角有个蓝色符号），双击之后应该是先下载再打开。这样的设计我个人感觉应该是一个类似于“选择性同步”的功能，用户可以指定同步哪些文件或者不同步。但显示在资源管理器上大家肯定都会以为这是本地的正常文件啊！个人感觉还不如像已有的同步盘那样，不在本地的目录压根就不显示，然后选择性同步的功能放到软件菜单中。
**三、内链和外链**
分享功能我觉得还是比较令人满意的。内链分享需要搜人，而且不能搜学号，只能搜显示名（昵称）。里面权限控制挺精细的，反正比 Seafile 要好，唯一担心的就是不知道文件锁定功能做得怎么样。比方说俩人同时上传同名文件的冲突如何处理。外链功能也比较强大，可以让没有账号的人匿名下载，上传。用外链功能就可以轻松**收发作业（记得关闭下载和预览权限）**，在这之前数院有老师收作业是用数院 FTP 的，这下可以解放了。
**四、其他一些奇葩设定**
用户的显示名（昵称）是可以自己改的，而且别人用过的名字自己就不能用了。所以一定要**趁早抢注 ID 啊**，谁也不希望自己的 ID 叫 <u>张三123</u> 这种东西吧。每个用户都有一个和自己显示名同名的顶层个人文档，改完显示名后这个目录名字也会发生变化。
<font color="#c00000">**更新：9 月 27 日学校改动了配置，网页端已经不能改显示名了。在 Windows 客户端可以改，但改完会变回去，怀疑是 BUG。**</font>
建立新的组文档时，组文档的名字在全局也是唯一的。例如我建立了一个名为 data 的组文档别人就建立不了了。所以组文档名也要抢注啊。从内链的路径上来看，所有人都可以在 AnyShare:// 顶层空间下创建东西，这就难怪各种文档名字不能一样了。这种设定我还是第一次见，之间用过所有的同步盘都不存在这个问题。另一个奇妙的事情是组文档的文件名不能起成别人的显示名（虽然不会有人无聊到干这事就是了）。
**五、暂时没玩到的功能**
隔离区，不知道是干啥的。可能是不让上传有病毒或者是敏感文件吧。
审查功能感觉很高级，有需求的话会看一眼。
另：客户端附带的手册其实挺详细的，看一下总没坏处。
总之以上几点直接把我这个 Linux 用户劝退了。正式出了以后可能会偶尔用下分享功能，本来想着有了这个可以和校外的人更好合作改 paper，结果……还是滚回去用组里的 Seafile 吧（
